FIGMA Design

UX & UI Design Google Home Page

Recreating Chrome's Home Page Using Figma.

As part of my journey to become a better UX and UI Designer, learning new design tools becomes a necessary learning curve that one has to endure. Already being proficient in using Adobe's XD and Zeppelin and gaining some commercial experience using both sets of software too, there was another piece of software that other designer's were persuading me to learn called Figma. Figma on one hand, is similar to the aforementioned in the sense that - yes, you can design Wireframes, High Fidelity designs, design Vectors and make and share prototypes alike. However, Figma offers a feature that unlocks the boundaries that all of the other's cant escape from yet and that is responsive design using their new feature named Auto-Layer!

Please read on through this Case Study on recreating Google Chromes homepage using Figma - Thank You and please enjoy!

Case Summary

Product Vision

The vision for this case study was to establish a clear understanding on using Figma as a design tool to expand my knowledge in UI and UX Design. Focusing on:

  • Upgrading skills in UX & UI design using Figma's software to enhance break point design with responsiveness
  • An over arching aim to,"Investigate whether or not a design can be responsive in as less break points as possible."
  • To answer this, I had to research development that Figma has made to cater for responsive design.
  • Lastly, collect the data from this research and recreate a data driven responsive design of Google Chrome's Home Page.

Discovered User Problem

The issue that designers have whilst using Design software, is although their designs are very well structured, the software itself - in regards to Responsive Design is found to be either non existent or sub standard.
I intended to research a previously unexplored Design Software called Figma, having being influenced by other designers and intrigued from research conducted.
The aim was to add responsiveness to the usual break point design, so that developers could efficiently utilise on handover. It was discovered that although Figma did provide a tool called, 'Auto-Layer,' which encapsulated containers into a more flexible container for a more fluid control over responsiveness, it didn't cater completely and therefore a single page design was not possible at the time of writing (2021), rather the same breakpoint design as before but with responsiveness.

Outcomes

Tasked with researching and developing new skills in using a new Design Software, naming Figma, to create an interactive responsive break point design. Research discovered that Figma does have a feature that can cater for such a task as an integral part of their Software, they named, "Auto Layer." As the name suggests it automatically adds a layer to the selected items, which when used correctly, encapsulates them into a fluid child-container. Which, by itself add's constraints and enforces it to behave responsively either in a column or row on user interaction. This revolutionises design, allows the designer to work efficiently and concentrate more on the things that matter and allows the developer more insight into what the designer wants. This does add a further dimension to designing, when this is also added to the parent container, this cohesively make the responsive design possible.

It does have a major issue though. At the time of writing (2021), it was discovered that it's not completely the answer. As you will you see from this case study, the "Auto Layer," feature will only work up to a certain point, whereby the content inside is no longer responding effectively and a new breakpoint is required. Concluding that although having just one single interactive and responsive design was not possible yet. That being said it will make for more efficient designing and development having only needing two breakpoint designs which are also responsive and interactive.

Discovery Analysis

Carrying out Desk Research on the contenders with relatable Software on the market today for Responsive Design, moreover, which offered the best experience over all and which had the aforementioned Responsive Design feature. Below are the discovered 4 main contenders following the Desk Research and which products had the greater Statistical Deviation following the customer reviews in terms of User Experience:

  • Adobe XD
  • Figma
  • inVision
  • Affinity

Following the Customer Review research for each of the four contenders the following was scrutinised from various forums and online product reviews:

Adobe XD

Adobe XD whilst certainly the most successful for Longevity by consistently adapting and updating their product and community base, an equal amount of good and bad reviews were discovered through out.
The Pros: Simplistic UI, Low Learning Curve, Sharing And Cross Platform Capabilities, 'Repeat Grid' Makes Designing In This Respect Most Efficient.
The Cons: No Security To Protect The Mock Ups When Sharing, No Tools For Animating Elements (Key Frames), Poor Video Quality When Prototyping. ❌ No Fluid Design Feature.

Figma

Figma is a vector graphics editor and prototyping tool which is primarily web-based, with additional offline features enabled by desktop applications for macOS and Windows.
The Pros: Low Learning Curve, Simplistic UI, Cross Platform aAnd Collaboration, Auto Layout Prototyping.
The Cons: Both Web And Software Are Memory Load Heavy, No Built In Animation Tools (Keyframes), Converted CSS Code Is Below Standard. ✔ Fluid Design Feature Found!

inVision

InVision is a prototyping, collaboration and workflow tool.
The Pros: Prototyping, Supports Gestures, Web Ready Transitions And Animations Available, Cross Platform And Collaboration, Plug-ins, Online Community, Full Version Control
The Cons:Confusing And Complex UI, Sharing Settings And Toolbar Are Not Always Intuitive, Poor HCI, Element Inspector Is Not Always Precise. ❌ No Fluid Design Feature.

Affinity

Affinity Designer is a vector graphics editor by Serif.
The Pros: Intuitive UI, Modern Gestures, Great Artistic Tools, Cross Platform
The Cons:Needs Paid Subscription To Export, Poor Customer Support, Complicated And Clunky, Steeper learning curve compared to other software. ❌ No Fluid Design Feature.

Software Analysis

The chosen Software from the Desk Research Analysis, is named Figma. It is available as a free or paid online version or a downloadable version, all versions come with the, "Auto Layer," feature included. Research found that it is this same feature that should cater for the interactive, dynamic and responsive design described in the overarching aim of this case study.

On first thought, one must feel a little uncertain as far as responsive web design is concerned and past technology fails. As viewports resize, elements within the design resize and move around the grid too, from a row structure to a column based structure - here is where ones skepticism may increase, as this shift occurs in design more breakpoints are required. The need for an unbiased and full investigation on this new technology that could indeed change the way we design for the future, could indeed answer the over arching aim or not.

Figma's Auto Layer Feature

Design tools have optimised repetitive manual work, since the dawn of time. Nowadays we're experiencing new technology that allows us to focus more about the things that matter, that repeatedly go over slight adjustments if something needs to be added or removed from an element of a design. This is were my research discovered Auto Layer by Figma.

This feature allows frames to automatically resize according to their content, based on rules the designer sets. Objects could already respond to the Frame that contains them with Constraints, but now Frames can also react to objects within them through Auto Layout. For example, if one wanted to add content to a button. Before this technology was available, a designer would have to add width, padding and margins to their designs. Following the research, it was found that a designer can now effortlessly add an Auto layer to the button, which increases/decreases the constraints inside and outside of that element, depending on the work carried out - thus saving time and making the dynamic work.

Further research found that in order to make Auto Layer work, a group of elements must be selected, then Auto Layer can be selected, either by right clicking the option to do so or by selecting it in the options in Figma's GUI. Doing so, creates a Frame, this frame then became the parent container, naturally those inside would then become the children elements. Which would have been in the button above case - the button container and the text respectfully.

On application, it was quickly discovered that once Auto Layout was applied, many properties were converted. Padding, the spacing between children, the axis for lining up children, and the order of children are all guessed based on an average. If you apply to a group or a selection it becomes a frame. Any background layer disappears as frames can now have fill colors. Finally, text boxes that had a fixed size convert to auto, as Figma believe thatâ€™s what most people expect in an Auto Layout.

If you have Web Development experience, like myself, you may find that this is giving the designer the power to add a CSS Layout Module called Flexbox to their designs. Those that don't have Web Development experience, basically a Flexbox Layout module gives components on a web page a responsive structure enabling whole pages to have responsiveness as screen sizes decrease and increase, which in turn improves the structural integrity of a website/ application to be displayed on any device.

Wireframes

In order to replicate the re-design of the selected Google Home page, wireframes were produced, of which recorded the dimensions of the actual Google Home page at three main break points. Large Desktop (1800px), Tablet (780px) and Mobile Device (400px). The aim here was to record margins, padding and position of all the elements as the page was resized. This was to demonstrate, once the redesign was complete, whether using this technique would (a) resize and position elements within a reasonably accurate amount (within 10px) from the original specifications recorded and thus (b) answer the overarching aim of accurately make a responsive and interactive and responsive break point design.

After studying the Home page closely, examining each component's constraints (margin, padding, height and width) for any alterations and the only major difference when resizing, apart from obviously the width and height constraints of the viewport, was the empty container size (see wireframes above). Everything else remained consistent throughout. That being said, this change from lengthy vertical shift to a short horizontal space, could prove too difficult for this new feature. Of course at this point, this was just a presumption and nevertheless the test continued.

Another detail was discovered through the research, Auto Later can only be applied to objects that have an equal amount of distance between one and other. It may look as though all of the navigation and search elements are equidistance but they are not when scrutinised on closer inspection. That being said, a work around could be possible.
