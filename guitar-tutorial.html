Voice Controlled Guitar Tutorial
Case Study

UX Research, UX & UI Design, Front End Development

The product

Just Speak Guitar Tutorials, is a fictitious website I created as part of my University Dissertation. Its whole purpose id to measure the feasibility of navigating a rich content website using a non AI Voice Command Algorithm as set out by my Institution. No one knew the answer to this and to find the answer part of the task was to design the website using UX principles, find a Javascript library that could cope with such a task, test users both on a design case scenario, the what, the how and the why. Build the product then study the Quantitative and Qualitative values to calculate the overall aim.

The problem

A complete R and D project to investigate if such a voice command system exists, Design and Develop the website using UXC principles.

  • Desk Research
  • Survey What the user expects from the site
  • Test the prototypes using User Interviews and Testing
  • Find issues with the designs, iterate and test again
  • Build the user friendly site aimed at the given demographic Website using HTML, CSS and JS
  • Test the Website on the public domain
  • User friendly site aimed at the given demographic

The Task

Once the Project management was complete with the individual structured tasks at hand, the project started 1/1/2019 it had a deadline for 18/05/2021. This project was complete 3 weeks before the deadline with the over arching aim answered with a success, combined with various other projects through out the 3 years, earned myself a First Class BSc(hons) in Web Design and Web Development.

Desk Research

I studied a few similar applications available to identify patterns. so that the new site, although unique to the company, would be familiar to the user too. It was identified that none of these sites had a gallery.

  • Artyom.js
  • Anyang.js
  • Voix.js
  • PocketSphinx.js

After testing the capabilities of these libraries a decision was made to use Annyang.js; it was the smallest library available, had no AI technology and was the cut above the rest in relation to responsiveness and adaptation.

User Survey and Testing

To determine what the user expected from such a project, I conducted a user survey through Google Survey and a social media platform to get a wide prospective from the public domain and then applied that data to the designs. Below is the mobile version of the final iteration form testing.

User Testing - Round 1

Using User Interviews and Think Aloud Protocol, I launched a small test using the above Prototype to identify what the user's needs were, if there were any flaws and luckily there was only a slight flaw in color contrast!

Feedback

  • Simple HCI and navigation on all View ports.
  • Voice Command button to be available to the public.
  • Voice command to be displayed in a table as shown in the diagram, but hidden until needed.
  • Simplistic flow through out the pages.
  • Rich images.
  • Aesthetically pleasing layout.
  • A collective theme through out

User Testing - Round 2

Once the website was built a new set of volunteers tested the website.

Using User Interviews and Think Aloud Protocol, I launched another small test with the exact same set of tasks, however with a different set of Users. This was to create an un bias set of results and indeed to see if the concluding iteration was the most successful, improved comprehension, encapsulated the user emotions whilst keeping the cognitive load low whilst navigating through the site and completing the tasks.

Feedback

  • The overarching aim was calculated by triangulating the data from the surveys and the user testing and it was a success.
