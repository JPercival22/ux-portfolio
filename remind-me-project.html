Remind Me
Case Study

UX Research, UX & UI Design,User Testing

The product

Remind Me is a due to be deployed soon. The application is a user editable reminder application which allows the user to interact and engage fully to edit, delete, create and set reminders for their daily tasks. It has two versions a free version with generic allowances and a Premium version which has more advanced capabilities.

The problem

The company wants a Website to be designed that:

  • Defines their Brand
  • Advertises both versions of their product using their market research
  • Includes a gallery of images collected from their application
  • A review section with the provided results from their user testing of their product
  • Company contact section
  • e-commerce order section
  • User friendly site aimed at the given demographic

The Task

Design a website for the development team, that advertises their product. Identify what the User wants to see and what is important to show them.

Competitive Research

I studied a few similar applications available to identify patterns. so that the new site, although unique to the company, would be familiar to the user too. It was identified that none of these sites had a gallery.

Sketches

After completing my initial research, I sketch ideas. This allows for many leading iterations through varying design options.

User Testing - Round 1

Using User Interviews and Think Aloud Protocol, I launched a small test using the Wire Frame Prototype to identify what the user's needs were:

Feedback

  • Simple HCI and navigation on all View ports.
  • Combined Feature and Price list.
  • A more interactive and aesthetically pleasing review section, rather than a wall of text.
  • Simplistic Contact Form
  • Simplistic checkout.
  • Aesthetically pleasing checkout
  • A collective theme through out

Feedback

  • Having changed some of the buttons from a Primary button to a clickable text link instead, it was found that most user's were no longer having cognitive overload, now they could differentiate themselves what was a primary and a secondary button themselves.
  • The layout of the site has as a minimum amount of pages possible, to increase page loads on weaker networks.
  • User's were intrigued at the Review section, they interacted more with the carousel and the eye tracker found that during the test, 80% of the user's spent more time focusing on that area,which would be fair to infer full engagement and are reading the reviews too.
  • 90 % of the user's were able to navigate fully through the Designs, were able to hit the end point on the Checkout section, inferring an aesthetically pleasing and simplistic layout was achieved.
  • A blue theme was added through out the designs, using the psychological factors of blue as being sincere, trustworthy and a calm color.
