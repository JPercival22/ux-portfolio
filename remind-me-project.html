Remind Me
Case Study

UX Research, UX & UI Design,User Testing

The product

Remind Me is a due to be deployed soon. The application is a user editable reminder application which allows the user to interact and engage fully to edit, delete, create and set reminders for their daily tasks. It has two versions a free version with generic allowances and a Premium version which has more advanced capabilities.

The problem

The company wants a Website to be designed that:

  • Defines their Brand
  • Advertises both versions of their product using their market research
  • Includes a gallery of images collected from their application
  • A review section with the provided results from their user testing of their product
  • Company contact section
  • e-commerce order section
  • User friendly site aimed at the given demographic

The Task

Design a website for the development team, that advertises their product. Identify what the User wants to see and what is important to show them.

Competitive Research

I studied a few similar applications available to identify patterns. so that the new site, although unique to the company, would be familiar to the user too. It was identified that none of these sites had a gallery.

Sketches

After completing my initial research, I sketch ideas. This allows for many leading iterations through varying design options.

Wire frames

Once a User Flow and a successful round of testing the User reaction and needs have yielded sufficient information, I graphically design the High Def Designs using Computer Design Software. My preferred choice is using Adobe XD although I am proficient in using inVision, Zeppelin and most other Design software too. Images and Vectors are created in Illustrator and Photoshop.

User Testing - Round 1

Using User Interviews and Think Aloud Protocol, I launched a small test using the Wire Frame Prototype to identify what the user's needs were, to determine weather this first draft design had a credible User Flow, by recording the user interaction and following the natural flow of actions the User expected to happen on interacting:

Feedback

  • Simple HCI and navigation on all View ports.
  • Combined Feature and Price list.
  • A more interactive and aesthetically pleasing review section, rather than a wall of text.
  • Simplistic Contact Form
  • Simplistic checkout.
  • Aesthetically pleasing checkout
  • A collective theme through out

User Testing - Round 2

After studying the results from the first round it was discovered, in combination with the User Survey conducted during the first round, that users wanted a vertical scroll main index (home) page and by studying their interaction was able to design the initial User Flow. This is displayed below.

Using User Interviews and Think Aloud Protocol, I launched another small test with the exact same set of tasks, however with a different set of Users. This was to create an un bias set of results and indeed to see if the concluding iteration was the most successful, improved comprehension, encapsulated the user emotions whilst keeping the cognitive load low whilst navigating through the site and completing the tasks.

Feedback

  • Having changed some of the buttons from a Primary button to a clickable text link instead, it was found that most user's were no longer having cognitive overload, now they could differentiate themselves with what was a primary and a secondary button themselves. This was completed by using a pill button as the Primary Button on the Index page and using a designed camera icon for the secondary.
  • The layout of the site has as a minimum amount of pages possible, to increase page load speeds on weaker networks and page refreshes.
  • User's were intrigued at the Review section, they interacted more with the carousel and the eye tracker found that during the test, 80% of the user's spent more time focusing on that area, which would be fair to infer full engagement and are reading the reviews too.
  • 90% of the user's were able to navigate fully through the Designs, were able to hit the end point on the Checkout section, inferring an aesthetically pleasing and simplistic layout was achieved. At this point it was concluded that the minority of the participants failing the test didn't fully engage and were either willfully obstructing a fair result or were simply not bothered.
  • A blue theme was added through out the designs, using the psychological factors of blue as being sincere, trustworthy and a being an overall calm color, with high contrast text color of black on a White or Gainsboro background for the Visual Hierarchy.
